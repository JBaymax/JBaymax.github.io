PhotoShop安装图解

Posted on 2016-06-17   |     |   visitors

做为一个曾经做UI的程序媛来说，每次安装Adobe的软件都很麻烦，因为需要破解。所以就根据自己的情况，分享一下自己的安装经历，安装过程如下。

方法/步骤

1．把计算机的wifi或者宽带线缆暂时断开（*），因为安装过程中会要求你登录账号，这个过程会非常慢，所以我们不登登录，无网络的链接它会提示跳过登录。

2．把下载的安装包解压出来，双击文件夹内的“set-up.exe”,进入应用安装界面。

3．单击选择下方的“试用”，进入下一步，选择“以后登陆”。

4．选择安装路径，建议安装在系统盘以外的盘，如果那你系统盘足够大的话，安装在系统盘也是没有任何问题的。

5．接下来需要的就是你耐心的等待软件安装完成，正常的话四五分钟就可以安装完成，安装时间随电脑配置的高低而不同。

6．好了，到了这一步我们就已经安装成功了，单机对话框下方的“关闭”，进行破解。

7．破解方法也是很简单的，首先下载个破解补丁，下载解压完的文件如图所示，选择想用系统的补丁进行破解。（最下方会提供破解补丁网盘地址）。

8．以记事本的方式打开破解补丁的host，复制里面的全部内容，粘贴到电脑C盘Hosts里，保存后退出。Hosts文件地址为“C:\WINDOWS\system32\drivers\etc”。

9．完成上一步后，把破解补丁中的“amtlib.dll”复制到photoshop cc2015的安装路劲中来替换原来的“amtlib.dll”（建议替换时备份一下原来的amtlib.dll文件）。

10．如果上面的步骤你都完成的的话，那就大功告成了，破解完成了，我们打开PS然后退出，再也不会弹出那个烦人的试用窗口了，接下来就尽情的试用吧！

11．PhotoShop cc2015 64位主程序&破解补丁网盘链接：http://pan.baidu.com/s/1o6ACnyy 密码：o6qt。

配置Java的环境变量

Posted on 2016-06-03   |   In Java   |     |   visitors

Java的简单介绍

Java是由Sun公司开发的一种应用于分布式网络环境的程序设计语言，Java语言拥有跨平台的特性，它编译的程序能够运行在多种操作系统平台上，可以实现“一次编写，到处运行”的强大功能。

前期准备

系统环境：Win7
工具：JDK（安装JDK，才能开始配置的，这里就不讲如何安装的了。）

配置步骤：

  1. 右击 “我的电脑”—>“属性”—>“高级系统设置”—>“高级”—>“环境变量”:
  2. 找到“系统变量”，选择“新建”—>弹出“新建系统变量”对话框，在“变量名”文本框输入JAVA_HOME,在“变量值”文本框输入JDK的安装路径，切记安装路径后面不要加分号，单击“确定”按钮：
  3. 在“系统变量”选项区域中查看PATH变量，如果不存在，则新建变量 PATH，否则选中该变量，单击“编辑”按钮，在“变量值”文本框的起始位置添加%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin;或者是直接%JAVA_HOME%\bin;，单击确定按钮:
  4. 在“系统变量”选项区域中查看CLASSPATH 变量，如果不存在，则新建变量CLASSPATH，否则选中该变量，单击“编辑”按钮，在“变量值”文本框的起始位置添加.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar;。注意：如果CLASSPATH是新建的，那么在最前面要加.;
  5. 测试环境变量的配置是否成功。在DOS命令行窗口输入JAVAC或javac，不分大小写，输出帮助信息即为配置正确：

‘JAVAC’ 不是内部或外部命令解决方法

原因一、没有安装jdk，只安装了jre
解决方案：安装JDK
原因二：环境配置问题
解决方案：检查三处地方
第一 JAVA_HOME
JAVA_HOME的内容是jdk安装目录。如我的安装的位置D:\software\Development Tools\Java\JDK\jdk1.8.0_40，并且后边不带分号；
第二，检查CLASSPATH，这是一个jar包的调用。.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar;（前面有点号和分号，后边结尾也有分号。
或者可以写成“.;%JAVA_HOME%\lib”，效果是一样的；
第三，检查PATH变量是否正确
PATH变量很简单，就是jdk的bin目录的意思。添加%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin;或者是直接%JAVA_HOME%\bin;

问题

在安装时，总觉得哪里安装不对，所以想卸载重新安装，下面是从网上找的一些方法，可以先使用上面的方法查看版本号，在选择卸载的命令行。

  • 卸载

3.0.0版本执行npm uninstall hexo-cli -g，之前版本执行npm uninstall hexo -g。

  • 如何安装旧版本Hexo？

先卸载当前版本，以2.8.3为例，执行npm install hexo@2.8.3 -g，再初始化并安装依赖和插件。
