配置Java的环境变量

Posted on 2016-06-03   |   In Java   |     |   visitors

Java的简单介绍

Java是由Sun公司开发的一种应用于分布式网络环境的程序设计语言，Java语言拥有跨平台的特性，它编译的程序能够运行在多种操作系统平台上，可以实现“一次编写，到处运行”的强大功能。

前期准备

系统环境：Win7
工具：JDK（安装JDK，才能开始配置的，这里就不讲如何安装的了。）

配置步骤：

  1. 右击 “我的电脑”—>“属性”—>“高级系统设置”—>“高级”—>“环境变量”:
  2. 找到“系统变量”，选择“新建”—>弹出“新建系统变量”对话框，在“变量名”文本框输入JAVA_HOME,在“变量值”文本框输入JDK的安装路径，切记安装路径后面不要加分号，单击“确定”按钮：
  3. 在“系统变量”选项区域中查看PATH变量，如果不存在，则新建变量 PATH，否则选中该变量，单击“编辑”按钮，在“变量值”文本框的起始位置添加%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin;或者是直接%JAVA_HOME%\bin;，单击确定按钮:
  4. 在“系统变量”选项区域中查看CLASSPATH 变量，如果不存在，则新建变量CLASSPATH，否则选中该变量，单击“编辑”按钮，在“变量值”文本框的起始位置添加.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar;。注意：如果CLASSPATH是新建的，那么在最前面要加.;
  5. 测试环境变量的配置是否成功。在DOS命令行窗口输入JAVAC或javac，不分大小写，输出帮助信息即为配置正确：

‘JAVAC’ 不是内部或外部命令解决方法

原因一、没有安装jdk，只安装了jre
解决方案：安装JDK
原因二：环境配置问题
解决方案：检查三处地方
第一 JAVA_HOME
JAVA_HOME的内容是jdk安装目录。如我的安装的位置D:\software\Development Tools\Java\JDK\jdk1.8.0_40，并且后边不带分号；
第二，检查CLASSPATH，这是一个jar包的调用。.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar;（前面有点号和分号，后边结尾也有分号。
或者可以写成“.;%JAVA_HOME%\lib”，效果是一样的；
第三，检查PATH变量是否正确
PATH变量很简单，就是jdk的bin目录的意思。添加%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin;或者是直接%JAVA_HOME%\bin;
